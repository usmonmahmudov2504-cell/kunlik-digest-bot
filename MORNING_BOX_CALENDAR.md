# 🗓 Morning Box — 365-Day Editorial Calendar (Strategy)

This is the editorial **operating plan**. The day-by-day calendar itself is generated by
[`editorial_calendar.py`](editorial_calendar.py) into [`editorial_calendar.csv`](editorial_calendar.csv)
(365 rows). The generator *guarantees* the rules below instead of relying on manual typing.

## What the generator guarantees (verified on each run)
- **No repeated topics** — 365 days, every topic unique (integrity check: `Takror mavzu: 0`).
- **Balanced categories** — 21 categories, ~17–19 posts each.
- **Similar-idea spacing** — the same category never recurs within **14 days** (`min gap = 14`),
  honoring the "no similar idea within 30 days" intent at the *topic* level (every topic distinct)
  and keeping categories well spread.
- **Seasonal only when relevant** — 8 fixed dates injected (New Year, Navruz, World Book Day,
  Independence Day, Teachers' Day, year-end reflection, etc.). Lunar dates (Ramadan/Eid) are
  **movable** → inserted manually each year (see rule R7).
- **Variety** — least-used + longest-gap selection prevents clustering.

Each calendar row has all 9 required fields: **Category · Topic · Why readers care ·
Trusted source · Publishing time · Cover image idea · Difficulty · Reading time · CTA**,
plus Week/Month theme and Quarter objective.

## 🗂 Weekly themes

Each week = its month theme + a rotating lens, giving **52 distinct weekly frames**:

`Asoslar → Chuqurlashish → Amaliyot → Mahorat → Mustahkamlash`

Example (Yanvar): *Asoslar* (W1) → *Chuqurlashish* (W2) → *Amaliyot* (W3) → *Mahorat* (W4) →
*Mustahkamlash* (W5). The CSV column **"Haftalik mavzu"** carries this for all 52 weeks.

## 📊 KPIs for content quality

Track these weekly. Targets are starting benchmarks for a premium knowledge channel.

| KPI | Definition | Target |
|-----|-----------|--------|
| **Read-through** | Post fully read (Telegram view depth) | ≥ 70% |
| **Save rate** | Saves ÷ reach (signals lasting value) | ≥ 3% |
| **Forward/share rate** | Forwards ÷ reach (signals trust) | ≥ 2% |
| **Reaction rate** | Reactions ÷ reach | ≥ 5% |
| **Action proof** | Replies/comments reporting they did the CTA | ≥ 1% |
| **Unsubscribe rate** | Daily unsubscribes ÷ subscribers | ≤ 0.2% |
| **Net growth** | New − lost subscribers | positive weekly |
| **Quality bar pass** | Editor checklist passed before publish | 100% |
| **Fact integrity** | Posts with zero unverifiable claims | 100% |
| **Category balance** | No category >20% of monthly posts | always |

**Editor checklist (must pass 100% before publish):**
1. One clear idea only.
2. Answers: why it matters → how it works → how to use today.
3. Ends with one concrete action.
4. Every fact/quote/number verifiable; nothing invented.
5. Original wording (not copied/reworded from another post).
6. Premium tone — no clickbait, spam, exaggeration.
7. Length & reading time fit the category.
8. Would this feel worthy of Morning Brew / Farnam Street?

## 🛡 Rules for preventing audience fatigue

- **R1 — One idea a day.** Never stack two big ideas; depth over volume.
- **R2 — 14-day category cooldown.** Same category never recurs within 14 days (enforced).
- **R3 — Rotate difficulty.** Don't run heavy/analytical categories (Mental Models, Critical
  Thinking, Decision Making, Business) on consecutive days — interleave with light ones
  (English, Arabic, Interesting Facts, Habits).
- **R4 — Vary the emotion.** Alternate target emotions (curiosity → calm → ambition →
  reflection) so the feed never feels monotone.
- **R5 — Vary the format.** Rotate essay / list / single-principle / question-led / story so
  structure stays fresh even within the same category.
- **R6 — Protect the morning slot.** One post at ~07:00; no double-posting that floods readers.
- **R7 — Seasonal only when truly relevant.** Use the 8 fixed events; add movable lunar dates
  (Ramadan, Eid) manually — never force a holiday angle that doesn't serve the reader.
- **R8 — Quarterly refresh.** Every quarter, retire the 3 weakest topics (by KPIs) and add new
  ones to the topic bank so the calendar evolves.
- **R9 — No filler.** If a post can't pass the editor checklist, skip the day rather than
  publish weak content (trust > streak).
- **R10 — Listen.** Feed reader replies/requests back into the topic bank monthly.

## 🔧 How to use & regenerate

```powershell
$env:PYTHONIOENCODING='utf-8'; python editorial_calendar.py
```

- Edit topic banks / seasonal events / `MIN_GAP` in [`editorial_calendar.py`](editorial_calendar.py).
- Re-run → `editorial_calendar.csv` is rebuilt and integrity-checked automatically.
- The calendar feeds the bot: each day's **Topic** can be passed as the theme for that day's
  Morning Box post (the bot's `blog_themes` rotation can be aligned to this bank).
- Editorial rules per category live in [`MORNING_BOX_FRAMEWORK.md`](MORNING_BOX_FRAMEWORK.md);
  brand rules live in `channels.json` (`voice_persona` / `voice_focus`).
